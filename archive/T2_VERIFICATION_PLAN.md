# T2 验证计划

## 任务概述
- **任务**: 命令行待办事项管理工具
- **场景**: 代码开发 (中等复杂度)
- **模式**: GSD vs non-GSD

## 验证测试用例

### 功能测试
1. 添加任务 (含优先级)
2. 删除任务
3. 标记完成
4. 按优先级排序
5. 数据持久化 (JSON)

### 边界测试
1. 空任务列表
2. 大量任务 (100+)
3. 特殊字符任务名
4. 重复添加

### 错误处理测试
1. 删除不存在的任务ID
2. 标记不存在的任务
3. 无效优先级输入
4. JSON文件损坏

## 验收标准验证清单

| 标准 | 验证方法 |
|------|----------|
| 1. 所有功能正常工作 | 端到端测试 |
| 2. 数据正确持久化 | 检查JSON文件 |
| 3. 命令行交互流畅 | 人工测试 |
| 4. 代码结构清晰 | 代码审查 |

## 验证证据收集

需保存:
- 运行LOG
- 生成的JSON文件
- 功能测试结果
- 代码质量评分证据
