# GSD vs 非GSD模式Agent表现对比实验设计

## 一、实验目的

系统性评估GSD（Get Shit Done）框架对AI Agent任务完成表现的影响，明确其在不同复杂度任务和应用场景中的实际效果差异与适用边界。

---

## 二、实验变量设计

### 2.1 自变量

| 变量 | 取值 | 说明 |
|------|------|------|
| GSD模式 | 使用 / 不使用 | 二元变量，决定Agent是否启用GSD框架 |

### 2.2 控制变量

| 控制变量 | 控制方式 |
|----------|----------|
| Agent型号 | 统一使用同一型号（如Claude Sonnet 4） |
| 基础能力配置 | 保持相同的工具权限、上下文窗口大小 |
| 硬件环境 | 同一计算环境，相同API endpoint |
| 任务描述 | 标准化需求描述，两组实验完全一致 |
| 实验时间 | 同一时段执行，避免模型版本更新影响 |
| 操作人员 | 同一实验人员执行所有任务 |

| **复杂** | 开放式问题解决，需要创意、规划与迭代 | 创意设计类任务 |

### 5.2 质量指标

| 指标名称 | 测量方式 | 评分标准 |
|----------|----------|----------|
| 结果准确率 | 产出结果与预期目标的匹配程度 | 0-100% |
| 完整性评分 | 任务要求覆盖的完整程度 | 1-10分 |
| 错误率 | 产出中错误或缺陷的比例 | 0-100% |

| 指标名称 | 测量方式 | 评分标准 |

## 八、数据分析方法

### 8.1 描述性统计

对每项指标计算：
- 均值 (Mean)
- 标准差 (SD)
- 中位数 (Median)
- 极值范围 (Range)

### 8.2 差异显著性检验

| 数据类型 | 检验方法 | 适用条件 |
|----------|----------|----------|
| 正态分布数据 | 独立样本t检验 | 样本量≥30，正态分布 |
| 非正态分布数据 | Mann-Whitney U检验 | 样本量较小或非正态 |
| 分类数据 | 卡方检验 | 频数数据比较 |

### 8.3 效应量计算

使用 Cohen's d 计算效应量：

```
d = (M₁ - M₂) / SD_pooled
```

效应量解释：
- |d| < 0.2：微小效应
- 0.2 ≤ |d| < 0.5：小效应
- 0.5 ≤ |d| < 0.8：中等效应
- |d| ≥ 0.8：大效应

### 8.4 多因素分析

采用 **双因素方差分析 (Two-way ANOVA)** 分析：
- GSD模式的主效应
- 任务复杂度的主效应
- 场景类型的主效应
- 交互效应

## 十、GSD优势表现与适用边界分析框架

### 10.1 优势表现维度

| 维度 | 评估要点 | 预期假设 |
|------|----------|----------|
| **任务复杂度** | GSD在何种复杂度下优势最明显 | 复杂任务优势更大 |
| **任务类型** | GSD在何种场景下优势最明显 | 结构化任务（代码开发）优势更大 |
| **质量维度** | GSD对哪些质量指标提升最显著 | 完整性、可用性提升显著 |
| **效率维度** | GSD对效率的影响（可能增加准备时间但减少迭代） | 复杂任务总时间减少 |

### 10.2 适用边界分析

| 边界类型 | 分析内容 |
|----------|----------|
| **任务规模边界** | 多大规模的任务GSD开始体现优势 |
| **任务类型边界** | 哪些任务类型GSD可能无优势或劣势 |
| **用户类型边界** | 不同技能水平用户的GSD收益差异 |
| **资源消耗边界** | GSD的额外资源消耗是否可接受 |

### 10.3 边界判断标准

```
GSD优势指数 = (GSD得分 - 非GSD得分) / 非GSD得分

优势指数 > 0.2：GSD明显优势
0 < 优势指数 ≤ 0.2：GSD轻微优势
-0.2 ≤ 优势指数 ≤ 0：GSD轻微劣势
优势指数 < -0.2：GSD明显劣势
```

## 十一、实验注意事项

### 11.1 实验控制

| 注意事项 | 具体要求 |
|----------|----------|
| 重复实验 | 每组任务重复执行3次，取平均值 |
| 随机顺序 | 任务执行顺序随机化，消除疲劳效应 |
| 环境隔离 | 两种模式使用独立环境，避免交叉污染 |
| 盲评设计 | 评估者不知道产出来自哪种模式 |

### 11.2 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 任务中断 | 记录中断原因，重新执行该次实验 |
| 环境错误 | 排查环境问题，修复后重新执行 |
| 结果异常 | 标记异常数据，分析原因后决定是否剔除 |
| 评分争议 | 引入第二评估者，取平均分 |

### 11.3 数据质量保证

- 完整记录所有原始数据
- 建立数据备份机制
- 定期检查数据完整性
- 保留实验过程截图/日志

### 11.4 伦理考虑

- 明确告知实验目的
- 保护实验数据隐私
- 如实报告实验结果（包括负面结果）

## 附录B：统计检验代码模板

```python
import pandas as pd
from scipy import stats
import numpy as np

def analyze_gsd_effect(data_path):
    """
    分析GSD效应
    
    Parameters:
    -----------
    data_path : str
        实验数据文件路径
    
    Returns:
    --------
    dict : 包含统计检验结果
    """
    df = pd.read_csv(data_path)
    
    results = {}
    
    # 分组
    gsd_group = df[df['mode'] == 'GSD']
    non_gsd_group = df[df['mode'] == '非GSD']
    
    # 对每个指标进行检验
    metrics = ['completion_time', 'accuracy', 'completeness', 
               'satisfaction', 'token_usage']
    
    for metric in metrics:
        # 正态性检验
        _, p_gsd = stats.shapiro(gsd_group[metric])
        _, p_non_gsd = stats.shapiro(non_gsd_group[metric])
        
        if p_gsd > 0.05 and p_non_gsd > 0.05:
            # 正态分布，使用t检验
            stat, p_value = stats.ttest_ind(gsd_group[metric], 
                                             non_gsd_group[metric])
            test_type = 't-test'
        else:
            # 非正态分布，使用Mann-Whitney U检验
            stat, p_value = stats.mannwhitneyu(gsd_group[metric], 
                                                non_gsd_group[metric])
            test_type = 'Mann-Whitney U'
        
        # 计算效应量
        pooled_std = np.sqrt(
            (gsd_group[metric].std()**2 + non_gsd_group[metric].std()**2) / 2
        )
        cohens_d = (gsd_group[metric].mean() - non_gsd_group[metric].mean()) / pooled_std
        
        results[metric] = {
            'test_type': test_type,
            'statistic': stat,
            'p_value': p_value,
            'significant': p_value < 0.05,
            'cohens_d': cohens_d,
            'gsd_mean': gsd_group[metric].mean(),
            'non_gsd_mean': non_gsd_group[metric].mean()
        }
    
    return results

def two_way_anova(data_path, dv, factor1, factor2):
    """
    双因素方差分析
    
    Parameters:
    -----------
    data_path : str
        数据路径
    dv : str
        因变量列名
    factor1 : str
        第一个因素列名
    factor2 : str
        第二个因素列名
    """
    import statsmodels.api as sm
    from statsmodels.formula.api import ols
    
    df = pd.read_csv(data_path)
    
    model = ols(f'{dv} ~ C({factor1}) + C({factor2}) + C({factor1}):C({factor2})', 
                data=df).fit()
    anova_table = sm.stats.anova_lm(model, typ=2)
    
    return anova_table
```
