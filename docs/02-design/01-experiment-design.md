## 实验变量设计

### 自变量

| 变量 | 取值 | 说明 |
|------|------|------|
| GSD模式 | 使用 / 不使用 | 二元变量，决定Agent是否启用GSD框架 |

### 控制变量

| 控制变量 | 控制方式 |
|----------|----------|
| Agent型号 | 统一使用同一型号（Claude Sonnet 4） |
| 基础能力配置 | 保持相同的工具权限、上下文窗口大小 |
| 硬件环境 | 同一计算环境，相同API endpoint |
| 任务描述 | 标准化需求描述，两组实验完全一致 |
| 实验时间 | 同一时段执行，避免模型版本更新影响 |
| 操作人员 | 同一实验人员执行所有任务 |

## 任务复杂度梯度设计

### 复杂度定义

| 复杂度等级 | 特征描述 | 典型特征 |
|------------|----------|----------|
| **简单** | 单一步骤或少量步骤，目标明确，路径清晰 | 指令执行类任务 |
| **中等** | 多步骤逻辑推理，需要信息整合与判断 | 分析推理类任务 |
| **复杂** | 开放式问题解决，需要创意、规划与迭代 | 创意设计类任务 |

## 应用场景多样性设置

| 场景类型 | 场景特征 | 典型活动 |
|----------|----------|----------|
| **代码开发** | 结构化产出，可验证性强 | 完善需求、编写文档、编码、测试、验证 |
| **数据分析** | 信息处理密集，结论导向 | 需求理解、数据搜集、分析、归纳、报告生成 |
| **创意生成** | 开放性高，主观评价 | 决策支持、文案撰写、构思设计 |

## 评估指标体系

### 效率指标

| 指标名称 | 测量方式 | 单位 |
|----------|----------|------|
| 任务完成时间 | 从任务开始到最终交付的总时长 | 分钟 |
| 步骤数量 | 完成任务所需的总操作步骤数 | 个 |
| 资源消耗 | Token使用量、API调用次数 | Token/次 |
| 迭代次数 | 需要修正或重做的次数 | 次 |

### 质量指标

| 指标名称 | 测量方式 | 评分标准 |
|----------|----------|----------|
| 结果准确率 | 产出结果与预期目标的匹配程度 | 0-100% |
| 完整性评分 | 任务要求覆盖的完整程度 | 1-10分 |
| 错误率 | 产出中错误或缺陷的比例 | 0-100% |
| 可用性评分 | 产出可直接使用的程度 | 1-10分 |

### 过程指标

| 指标名称 | 测量方式 | 评分标准 |
|----------|----------|----------|
| 任务分解合理性 | 子任务划分的逻辑性与完整性 | 1-10分 |
| 资源调用效率 | 工具使用是否恰当、高效 | 1-10分 |
| 错误修正能力 | 遇到问题时的自我修正效率 | 1-10分 |
| 上下文管理 | 长任务中的上下文保持能力 | 1-10分 |

### 用户体验指标

| 指标名称 | 测量方式 | 评分标准 |
|----------|----------|----------|
| 指令符合度 | 最终产出与用户原始意图的匹配程度 | 1-10分 |
| 交互流畅度 | 人机交互的自然程度与响应质量 | 1-10分 |
| 结果满意度 | 用户对最终结果的整体满意程度 | 1-10分 |
| 信任度评分 | 用户对Agent可靠性的感知 | 1-10分 |

## 实验任务矩阵

### 任务分配表

| 任务编号 | 场景类型 | 复杂度 | 任务描述 |
|----------|----------|--------|----------|
| **T1** | 代码开发 | 简单 | PNG到WebP批量转换脚本 |
| **T2** | 代码开发 | 中等 | 命令行待办事项管理工具 |
| **T3** | 代码开发 | 复杂 | 个人财务管理系统 |
| **T4** | 数据分析 | 简单 | 销售数据分析 |
| **T5** | 数据分析 | 中等 | 电商用户评论分析 |
| **T6** | 数据分析 | 复杂 | 竞品竞争态势分析 |
| **T7** | 创意生成 | 简单 | 智能手表广告文案 |
| **T8** | 创意生成 | 中等 | 在线学习平台品牌设计 |
| **T9** | 创意生成 | 复杂 | 压力管理App产品方案 |

### 任务详情

#### T1: PNG到WebP批量转换脚本

**场景**: 代码开发
**复杂度**: 简单
**描述**: 编写一个Python脚本，实现将指定目录下所有PNG图片批量转换为WebP格式，保留原始文件名

**输入资源**: 测试PNG图片目录
**输出要求**:
- 格式: Python脚本
- 功能:
  - 递归遍历目录
  - 支持指定输出目录
  - 保留原始文件名结构
  - 可配置WebP质量参数

**验收标准**:
- 脚本可正常运行
- PNG文件正确转换为WebP
- 文件名保持一致
- 转换质量可配置

#### T2: 命令行待办事项管理工具

**场景**: 代码开发
**复杂度**: 中等
**描述**: 开发一个命令行待办事项管理工具，支持添加、删除、标记完成、按优先级排序、数据持久化存储功能

**输入资源**: 无特殊输入资源
**输出要求**:
- 格式: Python命令行工具
- 功能:
  - 添加任务（含优先级）
  - 删除任务
  - 标记任务完成
  - 按优先级排序显示
  - 数据持久化到JSON文件
  - 命令行界面友好

**验收标准**:
- 所有功能正常工作
- 数据正确持久化
- 命令行交互流畅
- 代码结构清晰

#### T3: 个人财务管理系统

**场景**: 代码开发
**复杂度**: 复杂
**描述**: 设计并实现一个个人财务管理系统，包含收支记录、预算管理、可视化报表、数据导入导出功能

**输入资源**: 无特殊输入资源
**输出要求**:
- 格式: 完整Python应用
- 功能特性:
  - 收支记录管理（增删改查）
  - 预算设置与跟踪
  - 分类统计与分析
  - 可视化图表展示
  - CSV数据导入导出
  - 数据持久化存储

**验收标准**:
- 所有核心功能实现
- 用户界面友好
- 数据安全可靠
- 文档完整

## 实验流程设计

### 实验准备阶段

```
┌─────────────────────────────────────────────────────────┐
│                    实验准备阶段                          │
├─────────────────────────────────────────────────────────┤
│  1. 环境配置                                             │
│     ├── 配置GSD环境（全局安装）                          │
│     ├── 配置非GSD环境（纯净Agent）                       │
│     └── 验证两种环境可用性                               │
│                                                         │
│  2. 任务准备                                             │
│     ├── 编写9组标准化任务描述                            │
│     ├── 准备任务所需输入资源                             │
│     └── 设计验收评分标准                                 │
│                                                         │
│  3. 评估工具准备                                         │
│     ├── 设计数据记录表格                                 │
│     ├── 准备计时工具                                     │
│     └── 配置过程记录方案                                 │
└─────────────────────────────────────────────────────────┘
```

### 单次实验执行流程

```
┌─────────────────────────────────────────────────────────┐
│                    单次实验执行流程                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Step 1: 任务初始化                                     │
│  ├── 记录实验编号、时间、环境信息                        │
│  ├── 加载标准化任务描述                                  │
│  └── 启动计时器                                         │
│                                                         │
│  Step 2: 任务执行                                       │
│  ├── 记录每个操作步骤                                   │
│  ├── 记录工具调用与响应                                  │
│  ├── 记录错误与修正过程                                  │
│  └── 监控资源消耗                                       │
│                                                         │
│  Step 3: 结果收集                                       │
│  ├── 保存完整输出结果                                   │
│  ├── 停止计时器                                         │
│  └── 记录Token消耗                                      │
│                                                         │
│  Step 4: 质量评估                                       │
│  ├── 按验收标准评分                                     │
│  ├── 填写评估指标表格                                    │
│  └── 记录异常情况                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 实验顺序设计

为消除顺序效应，采用**拉丁方设计**：

| 执行轮次 | 第1次 | 第2次 | 第3次 |
|----------|-------|-------|-------|
| 任务T1 | GSD | 非GSD | GSD |
| 任务T2 | 非GSD | GSD | 非GSD |
| 任务T3 | GSD | 非GSD | GSD |
| ... | ... | ... | ... |

每组任务重复执行 **3次**，交替使用GSD/非GSD模式。
